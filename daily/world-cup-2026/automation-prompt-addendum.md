# 世界杯日报自动化 Prompt 增补要求

生效时间：2026-07-06 17:20 CST（UTC+8）

本文件用于补充 `reporting-guidelines.md`、`source-policy.md` 和 ChatGPT 自动化任务中的世界杯日报执行要求。后续自动化每次运行时，除既有必读文件外，必须读取本文件。

## 1. 比赛日与北京时间窗口强制校验

每次生成或更新日报前，必须先完成以下三步：

1. 读取 FIFA / ESPN 赛程，或在固定来源不可稳定回读时读取可靠赛程源。
2. 将每场比赛的官方当地开球时间逐场换算为北京时间 / Asia/Singapore（UTC+8）。
3. 用换算后的北京时间日期判断：
   - 哪些比赛属于本期赛果窗口；
   - 哪些比赛属于“今日赛程 / 下一比赛日看点”；
   - 哪些比赛只能放入“后续赛程提示”。

不得把北美当地日期直接当成北京时间日期。不得因为赛事当地同一天就把比赛提前写进北京时间下一比赛日。

日报正文必须写清楚：

```markdown
官方比赛日：北美当地 YYYY-MM-DD。  
北京时间实际发生时段：YYYY-MM-DD HH:mm 至 YYYY-MM-DD HH:mm。  
赛程核查：以下赛程均已按 Asia/Singapore / UTC+8 换算；不属于该北京时间窗口的比赛不列入本期赛程。
```

如果存在跨日比赛，必须明确说明。例如：

```markdown
阿根廷 vs 埃及属于北京时间 2026-07-08 窗口，不列入北京时间 2026-07-07 下一比赛日。
```

## 2. 下一比赛日栏目写法

“今日赛程与关键看点 / 下一比赛日看点”只允许列出**紧接着本期之后的北京时间比赛窗口**。

例如：

- 北京时间 2026-07-06 日报之后，下一比赛日是北京时间 2026-07-07：葡萄牙 vs 西班牙、美国 vs 比利时。
- 阿根廷 vs 埃及、瑞士 vs 哥伦比亚属于北京时间 2026-07-08，不得写入 2026-07-07 下一比赛日。

如果要提示更远赛程，另设“后续赛程提示”，并明确日期。

## 3. xG 获取、补录与展示

xG 是重要参考，但不是主线。每次生成或更新日报时：

1. 先查 FBref / StatsBomb。
2. 若 FBref / StatsBomb 暂未更新或不可稳定取得，再查 FotMob xG。
3. 若 FotMob 页面能稳定回读 Top stats 或比赛页 xG，则使用 `FotMob xG`；若页面说明技术统计由 Opta 提供，可在说明中写作 `FotMob / Opta xG`。
4. 同一场比赛不得混用 FBref / StatsBomb 与 FotMob xG。
5. 不要让 xG 表长期为空；只有 FBref / StatsBomb 和 FotMob 都不可稳定取得时，才写“xG 暂无可靠数据”。
6. 如果日报已发布但后来在同一自动化运行或后续回读中稳定取得 xG，允许对既有日报做“xG 补录”，不得新建重复日报。
7. xG 补录时必须写明补录时间、来源和是否混用。

xG 表推荐写法：

```markdown
| 比赛 | 实际比分 | xG 来源与数值 | 简要解读 |
|---|---:|---:|---|
| 巴西 vs 挪威 | 巴西 1-2 挪威 | FotMob xG：巴西 2.61 - 挪威 1.05（FBref / StatsBomb 暂未稳定取得） | 巴西机会质量占优但输球，说明点球、门将和终结效率改变了结果。 |
```

## 4. 中文球队名与中文球员名

日报面向中文阅读者，正文应以中文名为主。

- 球队名尽量使用中文：法国、阿根廷、葡萄牙、西班牙、英格兰、巴西、挪威、摩洛哥、美国、比利时等。
- 球员名尽量使用中文译名；首次出现或小节标题出现时，在后面括号加原名：
  - 哈兰德（Erling Haaland）
  - 贝林厄姆（Jude Bellingham）
  - 哈里·凯恩（Harry Kane）
  - 姆巴佩（Kylian Mbappé）
  - 梅西（Lionel Messi）
  - C 罗（Cristiano Ronaldo）
- xG 表、赛果表、前三预测表也应使用中文球队名。
- 来源标题、URL 和媒体原文标题可以保留英文，避免破坏引用。
- 不确定中文译名时，优先采用常见中文译名；无法确认时保留原名，但不得在同一篇中中英文混乱切换。

## 5. 生成后自检清单

每次发布前必须逐项自检：

- [ ] 本期赛果是否全部属于同一个北京时间窗口？
- [ ] “官方比赛日”和“北京时间实际发生时段”是否同时写清？
- [ ] 下一比赛日是否只包含紧接着的北京时间窗口？
- [ ] 是否误把后天或更远比赛放入“下一比赛日”？
- [ ] 是否把当地日期直接当作北京时间日期？
- [ ] xG 表是否主动检查 FBref / StatsBomb 和 FotMob？
- [ ] 若 xG 可用，是否写明来源和数值？
- [ ] 若 xG 不可用，是否明确写明不可稳定取得，而不是空表？
- [ ] 球队名是否以中文为主？
- [ ] 球员首次出现是否采用“中文名（原名）”？
- [ ] 来源和数据覆盖说明是否交代赛程、赛果、xG、重大机会、球员评分、伤停和争冠模型状态？
- [ ] GitHub 远端日报、overview、README 是否已回读验证？

## 6. 失败或发现错误时

如果运行中发现历史日报或刚生成日报存在赛程、日期、xG 或命名错误：

1. 优先修正当前最新日报。
2. 对历史日报，如不宜大规模重写正文，应新增集中核查 / 更正记录，并在 README 挂入口。
3. 更新自动化 prompt 或本文件，防止重复错误。
4. 最终回复必须说明：发现的问题、修正方式、未修正原因、commit、回读验证结果。
