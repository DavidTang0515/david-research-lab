# 2026 世界杯日报规范

本规范适用于 2026 年世界杯赛事期间的每日动态分析。

## 1. 数据时点

- 自动化默认在北京时间每天 14:00 执行。
- 该时间用于覆盖北美当地最晚一档比赛：最晚场次可能在北京时间约 09:00 开球，淘汰赛如有加时和点球可接近中午结束，并为官方结果确认和分析预留时间。
- 每篇日报必须写明生成时间和数据截止时间。
- 只把已经结束并得到可靠来源确认的比赛写入“比赛结果”。
- 正在进行或尚未开始的比赛放入“今日赛程与看点”。
- 日期以北京时间为主要展示口径，同时说明赛事当地日期，避免跨时区混淆。

## 2. 来源优先级

1. FIFA 官方比赛中心、赛程、积分榜和赛事规则。
2. 各国家队官方信息。
3. Reuters、AP、BBC、The Guardian、ESPN 等可靠媒体的比赛报道。
4. Opta 等数据机构的比赛数据和模型。
5. 赔率只能作为市场预期辅助，不得当作事实或唯一排名依据。

当来源结果或统计不一致时，应保留不确定性并优先采用 FIFA 官方数据。

## 4. 赛果分析

每场重点比赛至少说明：

- 最终比分。
- 决定比赛走向的关键因素。
- 对本组积分、出线概率和淘汰赛落位的影响。
- 哪支球队的评价应上调或下调。

不要把控球率、射门数等单项数据直接等同于球队强弱。

## 5. 前三名动态判断

“最可能获得前三名的球队”指对本届赛事冠军、亚军和季军位置的动态预测，不是 FIFA 排名或当天积分榜。

每天必须列出三支球队，并分别说明：

- 预测位置：冠军 / 亚军 / 季军。
- 相比上一期：上升 / 下降 / 不变 / 首次进入。
- 当前依据：比赛表现、阵容健康、赛程路径、攻防稳定性和大赛容错。
- 最大风险。

判断应综合：

1. 已完成比赛的实际表现。
2. 小组出线与潜在淘汰赛路径。
3. 阵容深度、核心球员状态和伤停。
4. 攻防稳定性，而非只看单场大比分。
5. 可靠数据模型或市场预期，但不得照抄赔率。

小组赛期间不应因一场比赛剧烈改变长期判断，除非出现重大伤病、淘汰风险或明显结构性问题。

## 6. 48 队赛制影响

本届赛事由 12 个四队小组组成。各组前两名和八个成绩最好的第三名进入 32 强。

分析时应特别关注：

- 第三名跨组比较。
- 净胜球、进球数和纪律积分。
- 提前锁定小组第一对淘汰赛路径的影响。
- 强队是否为了避开特定半区而调整轮换。

## 7. 发布规则

每次更新应：

1. 创建或更新 `daily/world-cup-2026/YYYY-MM-DD.md`。
2. 更新本目录 `README.md`，将最新日报放在首位。
3. 保留历史入口。
4. 使用单一目标的提交说明并推送到 `main`。
5. 发布后回读日报和栏目首页。

如果当天没有已结束比赛，仍应更新赛程、伤停、晋级路径和前三名判断，并明确说明没有新增赛果。

## 8. 失败处理

若数据访问或 GitHub 同步失败，应说明：

- 哪些赛果已确认。
- 哪些数据尚未核实。
- 已检查的来源。
- 报告是否部分完成。
- GitHub 是否同步成功。
