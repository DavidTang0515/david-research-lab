# 2026 世界杯日报规范

本规范适用于 2026 年世界杯赛事期间的每日动态分析。

本栏目不是单纯赛果搬运，也不是纯赛后专栏。默认目标是：**动态日报 + 动态总览 + 赛后专栏式判断力**。写法以 `2026-06-24.md` 的完整框架为主要参考：先用一个清晰比赛日主线统领全文，再按赛果、精彩瞬间、格局影响、今日看点、争冠梯队、热门重估、前三名预测、亮眼球员、战术观察、结论、来源和数据覆盖说明展开。

---

## 0. 执行前必读

每次生成或更新世界杯日报前，必须先读取并遵守以下文件：

1. 仓库根目录 `AGENTS.md`。
2. 仓库根目录 `RESEARCH_PHILOSOPHY.md`。
3. `daily/world-cup-2026/reporting-guidelines.md`。
4. `daily/world-cup-2026/source-policy.md`。
5. `daily/world-cup-2026/README.md`。
6. `daily/world-cup-2026/overview.md`。
7. 上一期世界杯日报。
8. 自动化交接记忆中的世界杯日报要求。

其中，`source-policy.md` 是强制来源登记表。不得因为搜索便利自行增加或替换数据源；固定主、备来源均不可用时，必须写“暂无可靠数据”或说明“当前环境未稳定取得”，不能临时找陌生网站补数。

## 1. 数据时点

- 自动化默认在北京时间每天 17:00 执行。
- 部分西海岸晚场换算后可到北京时间 12:00 才开球；淘汰赛如有加时和点球，可能到 15:00 左右结束。17:00 为官方结果确认、积分更新和分析预留时间。
- 每篇日报必须写明生成时间、数据截止时间、官方比赛日和北京时间实际发生时段。
- 日期以北京时间 / Asia/Singapore 为主要展示口径，同时说明赛事当地日期，避免跨时区混淆。
- 只把已经结束并得到可靠来源确认的比赛写入“赛果”。正在进行或尚未开始的比赛只能放入“今日赛程与关键看点”。
- 如果用户要求“昨天到今天版”，覆盖窗口应明确写成“北京时间 YYYY-MM-DD 00:00 至本次生成时间”，并只整理该窗口内已经结束并确认的比赛。
- 如果同日多次更新，要说明是“增量回读”还是“完整窗口重写”，避免读者误以为新增了比赛。

## 2. 来源优先级

所有日报必须先读取并遵守 [`source-policy.md`](./source-policy.md)。该文件指定每类数据的主来源、固定备用来源和缺失处理方式。

固定口径如下：

- 赛程、比分、比赛状态、小组积分、晋级、淘汰、首发、换人、进球和红黄牌：FIFA 官方为主，ESPN 为固定备用。
- 基础技术统计：优先 FIFA；FIFA 缺失时整场使用 FotMob，不拼接两家数据。
- xG、重大机会和球员评分：仅使用 FotMob；缺失时不写。
- 阵容、伤停和停赛：国家队官方为主，FIFA、Reuters、AP 为备用。
- 比赛过程和战术背景：Reuters 为主，AP、BBC、The Guardian、ESPN 为备用。
- 争冠概率模型：Opta Analyst 为主；仅在其未更新且 Goldman Sachs 有明确公开更新时补充后者。
- 默认不使用赔率。

### 数据使用原则

- 每场重点比赛都应主动检查可获得的技术统计、首发与换人、关键球员、伤停和纪律信息。
- 优先核对射门、射正、预期进球（xG）、重大机会、门将扑救、红牌和关键换人。
- xG 等模型数据必须注明数据机构；不同机构口径不得直接混用。
- 控球率、传球成功率、角球和跑动距离只在能够解释战术或比赛走势时使用。
- 数据用于检验和约束判断，不为了显得完整而堆砌。
- 正文只展示对赛果、晋级形势、球队评价或前三名预测有解释力的数据。
- 无法取得可靠数据时，应明确说明，不用媒体形容词代替统计事实。
- 每篇日报必须在文末增加“数据覆盖说明”，交代赛果、技术统计、xG、伤停、Reuters 检查和争冠模型的实际来源与缺失情况。

## 3. 推荐日报框架

日报应优先采用以下框架。`2026-06-24.md` 是当前最完整样例。

```markdown
# 世界杯动态日报｜M月D日比赛日：一句话主线标题

生成时间：YYYY-MM-DD HH:mm CST（UTC+8，Asia/Singapore / 北京时间）  
数据截止：北京时间 YYYY-MM-DD HH:mm。  
官方比赛日：北美当地 YYYY-MM-DD。  
北京时间实际发生时段：YYYY-MM-DD 凌晨至上午，主要覆盖 X 组 / 淘汰赛已经结束并得到可靠来源确认的比赛。

## 开头总述

用 1-3 段写清楚本比赛日主线。不要只罗列比分，而要说明：谁升温、谁降温、谁回到主动区、谁出现结构性隐患、谁影响淘汰赛路径。

## 昨日及今晨赛果 / 比赛结果速览

| 小组 / 阶段 | 比赛 | 结果 | 直接影响 |
|---|---|---:|---|
| X | A vs B | 2-1 | A 进入主动区 / B 出局 / 第三名比较变化 |

## 最精彩的瞬间

### 瞬间标题，不只写进球

说明这个瞬间为什么重要：它改变了比赛叙事、球队心理、出线处境、淘汰赛路径或争冠评价。

## 对小组和晋级格局的影响

### 球队 / 小组：一句话判断

按重点球队或小组分段，说明积分、净胜球、最佳第三名、淘汰赛落位、强队评价和路径变化。

## 当前争冠梯队

变化：上升 / 下降 / 不变 / 未出战但相对位置变化。  
最大隐患：一句话写清楚。

### 阿根廷：一句话判断

变化：……  
最大隐患：……

### 葡萄牙：一句话判断

变化：……  
最大隐患：……

### 西班牙：一句话判断

变化：……  
最大隐患：……

### 英格兰：一句话判断

变化：……  
最大隐患：……

### 巴西：一句话判断

变化：……  
最大隐患：……

## 最可能获得前三名的球队

| 预测位置 | 球队 | 相比上一期 | 当前依据 | 最大风险 |
|---:|---|---|---|---|
| 冠军 | 法国 / 阿根廷 / 巴西等 | 上升 / 下降 / 不变 | 比赛表现、路径、阵容健康、核心状态、可靠数据 | 最大不确定性 |
| 亚军 |  |  |  |  |
| 季军 |  |  |  |  |

补充说明：解释为什么某支热门进入或掉出前三。

## 表现亮眼的球员

### 球员｜球队

不只写进球。说明他的战术价值、心理价值或结构意义，例如中锋牵制、边后卫纵深和宽度、爆点球员打破节奏、门将关键扑救、整体防线或中场组合。

## 战术观察

写球队如何赢、如何被限制，以及强队暴露的问题。重点关注低位防守、转换效率、边路推进、中路连接、肋部插入、禁区包抄、定位球、门将和换人。

## 今日结论

用 2-3 段总结本比赛日真正留下的判断。强调“结果”和“内容”要分开：大胜要看对手强度，0-0 或小胜要判断是结构性问题还是比赛管理。

本期暂无需要晋级的趋势信号。

## 来源

## 数据覆盖说明

- 小组积分、最佳第三名和淘汰赛路径：说明来源。
- 技术统计：列出取得统计的重点比赛。
- xG 可用性：说明是否采用 FotMob。
- 伤停来源：说明国家队官方、FIFA、Reuters 或 AP 的检查结果。
- Reuters 检查：说明哪些重点比赛取得了 Reuters 稿件，哪些没有。
- Opta 更新：说明是否有可核验的新概率。
```

## 4. 标题和开头写法

标题应尽量包含“比赛日主线”，不要只写日期。例如：

- `世界杯动态日报｜6月23日比赛日：葡萄牙大胜回血，英格兰被拖入泥潭`
- `世界杯动态日报｜昨日到今日版：阿根廷稳住路径，葡萄牙进入硬线`

开头总述必须回答三个问题：

1. 这个比赛日最重要的主线是什么？
2. 哪些球队的评价发生变化？
3. 这种变化是积分变化、路径变化、伤停变化，还是比赛内容变化？

---

## 5. 赛果分析

每场重点比赛至少说明：

- 最终比分。
- 决定比赛走向的关键因素。
- 能够支持上述判断的关键比赛数据。
- 关键球员、重要换人、伤停或停赛影响。
- 对本组积分、出线概率、最佳第三名和淘汰赛落位的影响。
- 哪支球队的评价应上调或下调。

非重点比赛可只写比分、关键球员和晋级影响；重点比赛再增加数据对照。具体展示哪些指标，由其解释价值决定。

不得把控球率、射门数等单项数据直接等同于球队强弱。强队大胜要判断对手强度是否足够；0-0 或小胜要区分是结构性问题、效率问题，还是小组赛管理。

---

## 6. 最精彩的瞬间

这一节必须选择真正影响比赛叙事、球队处境或赛事格局的瞬间，不只是罗列进球。

可选类型包括：

- 关键进球或补时绝杀。
- 老将纪录或核心复出。
- 门将关键扑救。
- 伤退、红牌或关键换人。
- 最后阶段横梁、门线解围或点球。
- 让球队从风险区回到主动区的节点。

每个瞬间必须写“为什么重要”。

---

## 7. 对赛事走向的影响

在“小组和晋级格局影响”基础上，进一步说明对以下内容的影响：

- 小组出线和头名竞争。
- 最佳第三名比较。
- 32 强或后续淘汰赛路径。
- 强队评价和夺冠格局。
- 球队心理状态和轮换空间。

可使用类似表达，但必须有事实支撑：

- “从风险区回到主动区”。
- “积分还可以，但内容有隐患”。
- “续命成功，但问题没消失”。
- “稳定派”。
- “路径变硬”。
- “结果合格，内容待验证”。

## 8. 夺冠热门重估

每天必须持续跟踪以下球队：

- 法国。
- 阿根廷。
- 葡萄牙。
- 西班牙。
- 英格兰。
- 巴西。

没有出战的热门，也可以基于上一期状态和本比赛日侧面影响做克制评估，但不得为了写满而过度发挥。

每支球队建议写：

- 当前判断。
- 相比上一期变化。
- 当前依据。
- 最大隐患。

对葡萄牙、英格兰、巴西等热门球队，要区分积分形势和比赛内容。

## 9. 前三名动态判断

“最可能获得前三名的球队”指对本届赛事冠军、亚军和季军位置的动态预测，不是 FIFA 排名或当天积分榜。

每天必须列出三支球队，并分别说明：

- 预测位置：冠军 / 亚军 / 季军。
- 相比上一期：上升 / 下降 / 不变 / 首次进入。
- 当前依据：比赛表现、阵容健康、赛程路径、攻防稳定性和大赛容错。
- 最大风险。

判断应综合：

1. 已完成比赛的实际表现。
2. 小组出线与潜在淘汰赛路径。
3. 阵容深度、核心球员状态和伤停。
4. 创造机会与限制机会的能力，包括可靠来源的 xG、射正和重大机会数据。
5. 门将表现、定位球效率、红牌和关键换人等比赛变量。
6. 攻防稳定性，而非只看单场大比分。
7. 可靠数据模型或市场预期，但不得照抄赔率。

小组赛期间不应因一场比赛剧烈改变长期判断，除非出现重大伤病、淘汰风险或明显结构性问题。淘汰赛阶段可以更快调整，但也必须说明原因。

## 10. 表现亮眼的球员

这一节不应只列进球者。应解释球员的战术价值、心理价值或结构意义。

可写对象包括：

- 中锋：牵制、压迫、抢点、背身连接。
- 边后卫 / 翼卫：纵深、宽度、回防覆盖。
- 爆点球员：一对一、转换、打破节奏。
- 门将：关键扑救、出球、点球或高球处理。
- 中场组合：节奏控制、压迫、二点球。
- 整体防线：低位防守、协防、限制核心球员。

## 11. 战术观察

战术观察重点写：

- 球队如何赢。
- 球队如何被限制。
- 强队暴露的问题。
- 换人是否改变比赛走势。
- 数据是否支持肉眼判断。

重点关注：

- 低位防守。
- 转换效率。
- 边路推进。
- 中路连接。
- 肋部插入。
- 禁区包抄。
- 定位球。
- 门将表现。
- 关键换人。

对强队必须区分“结果好”和“内容有说服力”。

## 12. 48 队赛制影响

本届赛事由 12 个四队小组组成。各组前两名和八个成绩最好的第三名进入 32 强。

分析时应特别关注：

- 第三名跨组比较。
- 净胜球、进球数和纪律积分。
- 提前锁定小组第一对淘汰赛路径的影响。
- 强队是否为了避开特定半区而调整轮换。
- 小组赛结束后，应切换为完整 32 强淘汰赛路径视角。

## 13. 发布规则

每次更新应：

1. 创建或更新 `daily/world-cup-2026/YYYY-MM-DD.md`。
2. 更新本目录 `README.md`，将最新日报放在首位。
3. 更新 `overview.md` 中的前三名预测、比赛阶段、小组出线、晋级、淘汰、最佳第三名和淘汰赛路径。
4. 保留历史入口。
5. 使用单一目标的提交说明并推送到 `main`。
6. 发布后回读日报、动态总览和栏目首页。

如果当天没有已结束比赛，仍应更新赛程、伤停、晋级路径和前三名判断，并明确说明没有新增赛果。

不要生成 draw.io、Mermaid、思维导图或静态路线图。

---

## 14. 失败处理

若数据访问或 GitHub 同步失败，应说明：

- 哪些赛果已确认。
- 哪些数据尚未核实。
- 已检查的来源。
- 报告是否部分完成。
- GitHub 是否同步成功。
- `overview.md` 和 `README.md` 是否已更新。

## 15. 世界杯结束后

世界杯结束后应生成一次赛事复盘，冻结 `overview.md` 为最终赛事总览，并提醒暂停或关闭自动化。
