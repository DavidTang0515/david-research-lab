# 2026 世界杯固定数据源

本文件锁定世界杯日报、世界杯周报和动态总览使用的数据来源与口径。除非用户明确要求修改本文件，否则每日自动化不得自行增加或替换来源。

`reporting-guidelines.md` 是写作和流程规范；本文件是数据来源和口径规范。两者冲突时，先以本文件确定来源，再按 `reporting-guidelines.md` 展示。

---

## 1. 固定来源矩阵

| 数据类别 | 主来源 | 固定备用来源 | 临时确认来源 | 使用规则 |
|---|---|---|---|---|
| 赛程、开球时间、比赛状态 | FIFA 官方比赛中心 | ESPN 比赛中心 | Reuters、AP、BBC、The Guardian | 必须先取得原始赛程并换算为北京时间 / Asia/Singapore（UTC+8）。不得用北美当地日期直接推断北京时间比赛日。主来源可访问时不得用媒体比分替代；若 FIFA/ESPN 当次运行中短时间内不可稳定回读，但权威媒体已发布明确终场赛果，可用于当期日报发布，并标注“待 FIFA/ESPN 回读确认”。 |
| 比分、晋级和淘汰 | FIFA 官方比赛中心 | ESPN 比赛中心 | Reuters、AP、BBC、The Guardian | 终场、加时和点球比分必须写清。淘汰赛晋级结果可在权威媒体明确确认后先用于日报路径更新，并标注待官方回读。 |
| 小组积分、净胜球、最佳第三名 | FIFA 官方积分榜及赛事规则 | ESPN 积分榜 | Reuters、AP | 最佳第三名和同分规则优先以 FIFA 为准；若临时采用 ESPN 或媒体口径，必须在数据覆盖说明中写明。 |
| 首发、换人、进球、助攻、红黄牌 | FIFA 官方比赛报告 | ESPN 比赛记录 | Reuters、AP、BBC、The Guardian | 关键事件冲突时以 FIFA 为准；官方暂不可用时可采用权威媒体明确记录，但必须标注来源。 |
| 射门、射正、控球率、扑救等基础统计 | FIFA 官方技术统计 | FotMob | 无 | 同一场比赛只采用一个来源的完整口径；不得把 FIFA 与 FotMob 基础统计拼接使用。 |

| 重大机会和球员评分 | FotMob | 无 | 无 | 重大机会和球员评分仍以 FotMob 为准；FBref 不替代重大机会和球员评分。缺失时不展示。 |
| 阵容、伤停、停赛和复出 | 国家队官方发布 | FIFA、Reuters、AP | Reuters、AP | 不采用未获确认的社交媒体传闻；影响前三预测前必须说明官方确认状态。 |
| 比赛过程和战术背景 | Reuters | AP、BBC、The Guardian、ESPN | 无 | 用于解释数据，不替代官方赛果；但 FIFA/ESPN 暂不可稳定回读时，Reuters/AP 可作为终场赛果临时确认来源。 |
| 争冠概率模型 | Opta Analyst | Goldman Sachs 已公开模型 | 无 | 必须注明更新时间；不直接照抄为最终判断。 |
| 市场预期 | 不设固定主来源 | 不设固定备用来源 | 无 | 默认不使用；只有解释市场预期时才临时引用并明确标注。 |

## 2. 每日必查来源

每天生成日报前，至少检查：

1. FIFA 官方赛程、赛果和积分榜。
2. ESPN 比赛中心或积分榜，作为 FIFA 暂不可回读时的固定备用。
3. FIFA 官方比赛报告或技术统计。
4. FBref / StatsBomb 对当日重点比赛的 xG，优先用于“实际比分 vs xG 对比”表。
5. 若 FBref / StatsBomb 暂未更新或不可稳定取得，再检查 FotMob xG；若采用 FotMob，应在表格和数据覆盖说明中明确标注。
6. FotMob 对当日重点比赛的重大机会和球员评分；FBref 不替代这两项。
7. 涉及前三名候选球队时，检查相关国家队官方伤停和阵容信息。
8. Reuters 对重点比赛或重大伤停的报道。
9. AP、BBC、The Guardian 对尚未稳定取得 FIFA/ESPN 回读的终场赛果进行交叉验证。
10. Opta Analyst 是否更新争冠概率。
11. 当前仓库里的上一期日报、`schedule-audit-2026-07-06.md` 和 `automation-prompt-addendum.md`，用于避免重复赛程窗口错误。

如某项没有更新，应记录“未更新”或“暂无可靠数据”。若 FIFA/ESPN 当次运行中不可稳定回读，但 Reuters/AP 等权威媒体已经明确确认终场赛果，可以发布日报，但必须在正文和数据覆盖说明中写明“赛果采用权威媒体临时确认，待 FIFA/ESPN 回读确认”。不得使用搜索结果摘要、百科、无数据说明的自媒体或社交媒体传闻补数。

## 3. 北京时间赛程口径锁定

世界杯日报的主展示时区为北京时间 / Asia/Singapore（UTC+8）。

- 每场比赛必须以官方开球时间为基础，换算到北京时间后再判断属于哪个日报窗口。
- “官方比赛日”可以写北美当地日期，但“赛果窗口”和“下一比赛日”必须按北京时间判断。
- “下一比赛日 / 今日赛程”只允许列紧接着本期之后的北京时间比赛窗口。
- 后天或更远比赛不得提前写入下一比赛日；如确有必要提示，另设“后续赛程提示”。
- 若赛程信息来自非 FIFA / ESPN 的临时来源，必须在数据覆盖说明中写明“待 FIFA / ESPN 回读确认”。

示例：北京时间 2026-07-06 日报之后，下一比赛日是北京时间 2026-07-07 的葡萄牙 vs 西班牙、美国 vs 比利时；阿根廷 vs 埃及、瑞士 vs 哥伦比亚属于北京时间 2026-07-08，不得写入 2026-07-07 下一比赛日。

## 4. 数据口径锁定

- 比分、积分、晋级、纪律和比赛事件：优先 FIFA 口径；FIFA 暂不可回读时使用 ESPN；FIFA/ESPN 均不可稳定回读但 Reuters/AP 等权威媒体明确确认终场赛果时，可作为当期临时发布口径，并标注待官方回读。
- 基础技术统计：优先 FIFA；FIFA 缺失时整场切换为 FotMob，不拼接两家数据。
- xG：FBref / StatsBomb 为主；FBref / StatsBomb 暂未更新或不可稳定取得时，可整场使用 FotMob xG。必须在表格和数据覆盖说明中明确写出口径，不得混用。
- 重大机会和球员评分：仅使用 FotMob；FBref 不替代这两项。
- 争冠概率：优先 Opta Analyst；只有 Opta 没有更新且 Goldman Sachs 发布了明确模型更新时，才可补充后者。
- 不使用搜索结果摘要、百科和无数据说明的自媒体作为分析数据源。

---

## 5. xG 展示与补录规则

- 每场已结束重点比赛都应主动检查 xG。
- 若 FBref / StatsBomb 可稳定取得，使用 `FBref / StatsBomb xG`。
- 若 FBref / StatsBomb 暂未稳定取得但 FotMob 可稳定取得，使用 `FotMob xG`。
- 若两个来源都不可稳定取得，才写 `xG 暂无可靠数据`。
- 同一场比赛不得混用两种 xG。
- 不要让 xG 表长期为空；已发布日报后续拿到固定来源 xG 时，可以更新原日报做 xG 补录。
- xG 补录必须写明补录时间、来源和是否混用。
- xG 只作为比赛过程辅助，不得贯穿全文，不得把争冠判断写成 xG 专题。

---

## 6. 中文名口径

- 正文中的球队名尽量使用中文。
- 球员名尽量使用中文译名；首次出现或作为小节标题时，采用“中文名（原名）”。
- xG 表、赛果表、前三预测表也应使用中文球队名。
- 来源标题和 URL 可保留英文。
- 不确定中文译名时，优先使用常见译名；无法确认时保留原名，但不得在同一篇中频繁中英文混用。

---

## 7. 正文引用要求

每篇日报末尾增加“数据覆盖说明”，至少写明：

- 本期赛果和积分采用哪个来源；若采用 Reuters/AP 等权威媒体临时确认，必须写明待 FIFA/ESPN 回读确认。
- 本期赛程如何按北京时间换算；下一比赛日是否只列紧接着的北京时间窗口。
- 哪些重点比赛取得了技术统计。
- xG 是否可用以及采用哪个来源；若采用 FBref / StatsBomb，必须写明“未与 FotMob 混用”；若采用 FotMob，必须写明“FBref / StatsBomb 暂未稳定取得”。
- 重大机会和球员评分是否可用；缺失时明确写明 FotMob 未稳定取得。
- 伤停信息来自哪些国家队官方渠道。
- 争冠模型是否有当日更新。

正文中的关键数字应在相应段落或来源列表中标明来源。不能只在文末列网站名称，却不说明数字来自哪里。

---

## 8. 来源不可用时

- FIFA 暂不可用：优先使用 ESPN 确认比分和积分，并标注“待 FIFA 回读确认”。
- FIFA/ESPN 均暂不可稳定回读：若 Reuters、AP、BBC、The Guardian 等权威媒体已明确确认终场赛果，可用于当期日报发布，并标注“待 FIFA/ESPN 回读确认”。
- FBref / StatsBomb xG 暂未更新或不可稳定取得：使用 FotMob xG；若 FotMob 也不可稳定取得，才在表中写“xG 暂无可靠数据”。
- FotMob 重大机会和球员评分不可用：不使用其他来源替代。
- 国家队官方未确认伤病：写“伤情尚未得到国家队官方确认”，不据此调整前三名预测。
- Opta 未更新：沿用上一期模型基准并注明日期，不假设概率已经变化。

---

## 9. 修改规则

只有出现以下情况时才调整固定来源：

- 来源长期停止更新或无法访问。
- 数据口径发生重大变化。
- FIFA 发布新的官方数据平台。
- 用户明确要求更换。

调整时必须说明原来源、替代来源、口径差异和生效日期。
