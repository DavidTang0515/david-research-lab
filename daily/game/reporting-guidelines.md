# 游戏日报规范

本文件是 David Research Lab 总规则在游戏日报场景下的专项补充。发生冲突时，以仓库根目录 `AGENTS.md` 为准。

## 1. 定位

游戏日报负责记录当日经过核实的重要游戏信息，并提供初步影响判断和趋势线索。

日报不是新闻堆砌，也不是长期 Topic。它应回答：

1. 发生了什么？
2. 为什么值得关注？
3. 对玩家、平台、公司或产业有什么影响？
4. 国内外玩家主要关注什么？
5. 后续应该观察什么？
6. 是否出现值得进入 Research Lab 的结构性信号？

## 2. 范围

只收录与游戏直接相关的内容：

- 主机、PC、手游和独立游戏。
- 发售、延期、更新、补丁、DLC、试玩和评测。
- PlayStation、Xbox、Nintendo、Steam、Epic 等平台变化。
- 工作室、发行商、收购、裁员、财报、监管和版号。
- 电竞重大赛事和高影响争议。
- 与游戏直接相关的影视动画、音乐、周边和活动。
- 玩家社区高关注讨论。

排除泛科技、社会、政治、财经和娱乐新闻，除非它们对游戏产业有直接、明确影响。

默认无剧透，不披露关键剧情、结局、谜题答案和重要角色命运。

## 3. 来源优先级

1. 官方公告、平台博客、商店页面、补丁说明和监管文件。
2. 可靠游戏及行业媒体。
3. Steam、主机商店、评分聚合和官方赛事等一手平台数据。
4. 机核、B站、微博、贴吧、NGA、知乎、Reddit、ResetEra、X、YouTube 和 Steam 评论等社区来源。

国内线索优先关注机核的编辑、主播、节目和社区讨论，但不能只依赖单一站点。

来源冲突时，应写明分歧和不确定性，不得强行形成确定结论。

## 4. 选题标准

优先选择至少符合一项的内容：

- 直接影响大量玩家，如发售时间、价格、订阅、平台规则和重大补丁。
- 对产业有明确影响，如收购、裁员、监管、财报和发行政策。
- 涉及高关注游戏、工作室、创作者、系列或平台。
- 产生强烈且可核验的社区反应。
- 对平台竞争、商业模式或玩家消费方式具有持续观察价值。

避免用低影响预告片和零散传闻填充篇幅。

## 5. 覆盖窗口

- 使用“上一期日报生成后至本次生成时”的增量窗口。
- 正文必须写明生成时间和覆盖范围。
- 如跨越多日，应说明原因。
- 不无说明地重复上一期内容；确需跟进时，明确写出新增进展。

## 6. 日报结构

```markdown
# YYYY-MM-DD 游戏新闻日报

生成时间：
覆盖范围：

## 今日最重要

### 1. 标题

- 事件：
- 背景：
- 为什么重要：
- 可能影响：
- 国内态度：
- 国外态度：
- 后续观察：
- 来源：

## 其他值得关注

## 今日观察

## Research Lab 趋势信号

## 来源索引
```

“今日观察”应提炼共同主线，不重复新闻摘要。

## 7. 玩家态度

- 国内和国外态度分开描述。
- 区分共识、争议点、媒体评价和玩家反馈。
- 注明讨论来自哪些平台。
- 不以单条评论或小样本代表整体玩家。
- 媒体评测不能直接写成玩家共识。
- 证据不足时统一写：`暂未看到足够明确的玩家反馈`。

## 8. Research Lab 趋势信号

只有以下情况才提出趋势信号：

- 平台竞争或发行模式持续变化。
- 多家公司出现相同经营、裁员或投资方向。
- 玩家消费、订阅或内容偏好连续变化。
- AI、硬件、监管或宏观环境明显改变游戏产业。
- 同类信号连续多期出现。

每个趋势信号至少包含：

- 事件或变化。
- 已确认事实和主要来源。
- 时间属性：短期事件 / 周期变化 / 长期趋势。
- 影响强度和证据可信度：1—5。
- 建议去向：仅保留 Daily / Report / Topic / Dashboard。
- 判断理由和后续观察指标。

如果没有达到标准，写：`本期暂无需要晋级的趋势信号。`

日报自动化不得直接修改 Topic 或 Dashboard。

## 9. 发布规则

每次运行应：

1. 创建或更新 `daily/game/news-YYYY-MM-DD.md`。
2. 更新 `daily/game/README.md`，把最新日报放在 `Latest Reports` 首位。
3. 保留历史入口。
4. 有合格信号时创建 `daily/game/trend-signals/YYYY-MM-DD.md`。
5. 使用清晰、单一目标的 Commit。
6. 推送后回读远端日报和栏目首页，确认最终状态。

## 10. 失败处理

如果搜索、来源访问、脚本或 GitHub 写入失败，必须说明：

- 失败原因。
- 已检查来源。
- 已完成部分。
- 日报是否部分生成。
- GitHub 是否同步成功。
- 趋势信号是否生成。
