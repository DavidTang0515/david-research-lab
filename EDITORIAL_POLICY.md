# EDITORIAL_POLICY.md

# David Research Lab

## Editorial Policy v2.0

This document defines the long-term collaboration model of David Research Lab.

It supersedes previous assumptions that multiple AI assistants share equal editorial responsibilities.

---

# 1. Organization

## Research Director

David

Responsibilities:

- Decide research direction
- Decide research priorities
- Review final results
- Merge Pull Requests

---

## Chief Research Editor

Codex

Codex is the primary editor of this repository.

Responsibilities:

- Daily maintenance
- Information collection
- Weekly updates
- Monthly summaries
- Dashboard maintenance
- Topic maintenance
- Report maintenance
- Git operations
- Pull Request creation

Codex owns the day-to-day operation of the repository.

---

## External Editorial Board

ChatGPT

ChatGPT is not responsible for daily maintenance.

Instead, ChatGPT serves as an external editorial board.

Responsibilities:

- Strategic review
- Architecture review
- Trend analysis
- Quarterly review
- Major topic suggestions
- Repository quality assessment

ChatGPT should only participate when requested by the Research Director.

---

# 2. Daily Workflow

Normal workflow:

Research Director

↓

Codex updates repository

↓

Codex creates Draft PR

↓

Research Director reviews

↓

Merge

No ChatGPT participation is required.

---

# 3. ChatGPT Participation

ChatGPT should be invited only for:

- Major architecture adjustments
- Quarterly strategy review
- Annual review
- Repository quality review
- New Topic design
- Long-term trend analysis
- Editorial suggestions

ChatGPT should not be involved in routine weekly maintenance.

---

# 4. Editorial Principle

The repository is not a news archive.

Every update should transform:

News

↓

Trend

↓

Knowledge

↓

Methodology

Avoid simple event collection.

Focus on reusable long-term knowledge.

---

# 5. Repository Philosophy

Priority order:

1. Long-term Topics

2. Dashboard

3. Reports

Topics should evolve continuously.

Reports record history.

Dashboard reflects the current world state.

---

# 6. Weekly Operation

Codex should:

- Search important events
- Update weekly report
- Update Dashboard if necessary
- Update relevant Topics if long-term knowledge changes
- Submit Draft PR

Avoid unnecessary restructuring.

---

# 7. Monthly Operation

Codex should:

- Produce monthly summary
- Merge repeated observations into Topics
- Archive obsolete short-term information

Knowledge accumulation is more important than news accumulation.

---

# 8. Quarterly Review

Research Director may invite ChatGPT.

ChatGPT will review:

- Repository structure
- Topic quality
- Knowledge duplication
- Missing research directions
- Dashboard design
- Editorial consistency

Output should be an Editorial Review Report.

---

# 9. Pull Request Policy

Each Pull Request should focus on one objective.

Recommended size:

- One report
- One topic update
- One dashboard update

Avoid oversized Pull Requests.

---

# 10. Success Criteria

A successful repository is not the one with the most files.

It is the one that builds the most reusable knowledge.

The goal of David Research Lab is:

To continuously transform information into long-term research assets.
